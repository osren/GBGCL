# 服务器实验指南

> 上传修改后的代码到服务器，并在服务器上执行超参搜索实验。

---

## 一、需要上传的修改文件

本次 PHASE1 修改仅涉及 **1 个核心文件**：

| 文件 | 路径 | 修改说明 |
|------|------|---------|
| `src/train.py` | `GBGCL/src/train.py` | 新增 `--results_dir` 参数，支持自定义结果输出目录 |

其余文件（`sweepX.py`、`analyze_results.py`）本次未改动，但需要确保服务器上与本地版本一致。

---

## 二、上传到服务器

### 方式一：手动上传（推荐）

将以下 **2 个文件**通过 scp / rsync / SFTP 上传到服务器对应目录：

```
GBGCL/src/train.py          →  服务器 /path/to/GBGCL/src/train.py
GBGCL/tools/sweepX.py      →  服务器 /path/to/GBGCL/tools/sweepX.py
GBGCL/tools/analyze_results.py → 服务器 /path/to/GBGCL/tools/analyze_results.py
```

### 方式二：整个项目同步

如果服务器端是全新拉取的代码，只需确保 `src/train.py` 是最新版本即可，其他文件本次未改。

## 四、实验运行流程

### Step 1：Stage-A 粗筛

约 2–4 小时，150 epochs × 1 trial，快速探索最优超参组合。

```bash
cd /path/to/GBGCL

export SWEEP_STAGE=A
export SWEEP_WORKERS=2       # 并行数，根据服务器 GPU 数量调整
python tools/sweepX.py
```

### Step 2：查看粗筛结果

```bash
# 分析 Top 配置
python tools/analyze_results.py

# 结果输出到 analysis/overall_topk.csv
cat analysis/overall_topk.csv
```

### Step 3：Stage-B 精筛

根据 Stage-A 的 Top 结果（如有必要，手动更新 `sweepX.py` 中的 `FILTERS`），然后执行 700 epochs × 5 trials。

```bash
export SWEEP_STAGE=B
export SWEEP_WORKERS=1       # 精筛建议单进程，避免 GPU 竞争
python tools/sweepX.py
```

---

## 五、关键参数说明

| 环境变量 | 含义 | 推荐值 |
|---------|------|--------|
| `SWEEP_STAGE` | 实验阶段 | `A`=粗筛，`B`=精筛 |
| `SWEEP_WORKERS` | 并行进程数 | 粗筛 2，精筛 1 |
| `SWEEP_DEVICE` | 运行设备 | `cuda`（默认）或 `cpu` |

---

## 六、输出文件位置

| 文件 | 位置 |
|------|------|
| 每次 trial 的训练日志 | `logs/log_CUDA/<dataset>_<quity>_<sim>_<alpha>_<stage>.log` |
| 实验结果 CSV | `results/<dataset>_summary.csv` |
| Top 配置汇总 | `analysis/overall_topk.csv` |

---

## 七、注意事项

1. **断点续跑**：`sweepX.py` 会检查 `results/<dataset>_summary.csv`，已完成的组合会自动跳过，无需担心重复训练。
2. **Physics 数据集**较大（34K 节点），如遇显存不足（OOM），可将 `sweepX.py` 中 `HIDDEN_DIM` 降至 512，或将 `SWEEP_WORKERS` 降为 1。
3. **数据传输**：实验完成后，将 `results/` 和 `analysis/` 目录同步回本地进行分析。
