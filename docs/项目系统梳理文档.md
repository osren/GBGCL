## 1. 项目背景与定位

本项目全名 **GBGCL（Granular Ball Graph Contrastive Learning）**，是在 NeurIPS 2024 论文：

> *"Exploitation of a Latent Mechanism in Graph Contrastive Learning: Representation Scattering"* (SGRL)

的官方 PyTorch 实现基础上，引入**粒球计算（Granular Ball Computing）**思想进行的改进研究。

### 核心创新

| 创新点 | 说明 |
|--------|------|
| **粒球聚类模块** | 基于图拓扑与节点嵌入的质量函数，将图递归分裂成若干"粒球"（granular balls），每个球是一组语义相近的连通节点 |
| **球图扩散** | 在粒球层面构建球-球邻接矩阵，执行 K 步随机游走扩散，将平滑后的球心信息回写到节点 |
| **球级损失** | 球级散射损失（RSM 升维）+ 球级 InfoNCE（匈牙利对齐），在粒粒层面约束表示空间 |
| **BYOL 框架兼容** | 在已有 Online/Target 双网络结构上无缝叠加粒球模块，不破坏原有 SGRL 散射机制 |

### 3.2 模型定义 `models.py`

包含三个模型类：`Conv`、`Online`、`Target`。

#### `Conv`（图卷积编码器）

```
输入特征 x [N, F]
    ↓
GCNConv × num_layers（默认 1 层）
    ↓  PReLU + Dropout
节点嵌入 z [N, hidden_dim]
    ↓
投影头（Linear→PReLU→Dropout→Linear）
    ↓
投影表示 z_proj [N, proj_dim]
```

#### `Online`（在线网络，BYOL 主网）

核心职责：
1. 调用 `online_encoder` 获取原始嵌入 `h_1`
2. 通过归一化邻接矩阵执行 **num_hop 步散射**（全局跳传播）得到 `h_2`
3. 拼合 `h = h_1 + h_2` 后输入 `predictor`（两层 MLP）
4. 使用 EMA（动量 `momentum=0.99`）更新 `target_encoder` 参数

关键方法：

| 方法 | 说明 |
|------|------|
| `forward(x, edge_index)` | 返回 `(h, h_pred, h_target)`，即合并嵌入、预测向量、目标向量 |
| `embed(seq, edge_index, adj, Globalhop)` | 返回 `(h_1, h_2)` 原始嵌入 + 散射嵌入 |
| `update_target_encoder()` | EMA 动量更新 |
| `get_loss(z1, z2)` | 余弦相似度负均值（节点级 BYOL 损失）|

#### `Target`（目标网络）

- 共享 `target_encoder` 权重（由 Online 维护）
- `get_loss(z)`：正则化损失，防止嵌入坍缩（鼓励嵌入分散）：
  ```
  loss = -mean[(z - mean(z))² 的行和]
  ```

### 3.3 粒球聚类 `granular.py`

**核心类**：`Granular`

将整个图划分为若干**粒球（Granular Balls）**，每个球是一组拓扑连通且语义相近的节点集合。

#### 质量度量方式（`quity` 参数）

| 选项 | 说明 |
|------|------|
| `detach` | 基于模块化度（Modularity-like），考虑边权重与度期望 |
| `homo` | 基于节点嵌入相似度的同质性，相似邻居越多质量越高 |
| `edges` | 基于内部边密度与期望边数之差（归一化） |
| `deg` | 简单的边/节点比，轻量但信息量少 |

#### 节点相似度方式（`sim` 参数）

| 选项 | 说明 |
|------|------|
| `dot` | 点积（默认） |
| `cos` | 余弦相似度 |
| `per` | Pearson 相关系数 |

#### 核心算法流程

```
1. init_GB(graph)
   ├── 按度排序，选 sqrt(N) 个高度节点为初始球心
   ├── BFS 扩展，按相似度将节点分配到最近球心
   └── 返回初始子图、成员列表、质量值

2. split_bfs(graph, ...)           ← 递归分裂
   ├── 若节点数 ≤ 3：直接保留为球
   ├── 选取度最高的两个节点作为新球心
   ├── BFS 扩展，将节点按相似度分配到两个子球
   ├── 验证连通性（若子球不连通则停止分裂）
   ├── 计算两子球质量 qa, qb
   └── 若 quality_f > (qa + qb) / 2.5：保留当前球（不分裂）
       否则：对两子球递归调用 split_bfs

3. generate_GB(graph)
   ├── 若图连通：直接调用 get_GB_graph
   └── 否则：对每个连通分量分别调用，合并结果
```

#### 关键设计

- **分裂阈值**：父质量 vs 子质量均值的 2.5 倍，保证分裂确实提升了球内一致性
- **连通性保障**：分裂后子球必须连通，否则回退保留父球
- **初始球数**：`init_num = max(2, sqrt(N))`，自适应图规模

### 3.4 粒球工具 `gb_utils.py`

提供从"构球"到"扩散"到"损失"的完整功能链。

#### 核心函数一览

| 函数 | 功能 |
|------|------|
| `build_granules(node_embed, edge_index, quity, sim)` | 调用 `Granular` 构建粒球，返回球成员列表、球心、图结构 |
| `compute_ball_centers(node_embed, GB_node_list)` | GPU 向量化计算球心（成员嵌入均值），返回 `[B, d]` |
| `_build_ball_graph(...)` | 构建球-球邻接矩阵 `W~`（跨球拓扑边 + 球心相似度 + KNN 稀疏化） |
| `_diffuse_on_ball_graph(H0, W_tilde, beta, K)` | 在球图上执行 K 步扩散：`H^{t+1} = (1-β)H^t + β D⁻¹W H^t` |
| `granule_diffuse_and_write(...)` | **主函数**：构球 → 计算球心 → 构球图 → 扩散 → 回写节点 |
| `ball_scatter_loss(H_ball, ...)` | 球级散射损失（RSM 升维 + 角度阈值刹车 + 均匀分布正则） |
| `jaccard_between_balls(B1, B2)` | 计算两组球成员的 Jaccard 相似矩阵 `[B1, B2]` |
| `hungarian_matching(sim_mat)` | 最大化相似度的匈牙利匹配，返回 `List[(i,j)]` 配对 |
| `ball_infonce(Ha, Hb, pos_pairs, temp)` | 球级 InfoNCE 对比损失（正样本 = 匈牙利匹配对） |

#### 球-球图构建（`_build_ball_graph`）

```
权重融合模式（w_mode）：
  "topo"       → W = 跨球原图边数（拓扑权重）
  "center"     → W = relu(球心余弦相似度)（语义权重）
  "topo+center"→ W = topo_w + relu(sim_center)（融合，默认）

KNN 稀疏化：保留每个球的 Top-K 近邻，减少远距球干扰
加自环：W = W + I，保证扩散稳定性
```

#### 节点回写（残差融合）

```python
z_new[idx] = alpha_write * node_embed[idx] + (1 - alpha_write) * HK[b]
# alpha_write 控制原始嵌入保留比例（默认 0.5）
```

### 3.5 训练入口 `train.py`

#### 主函数 `run(args)`

```
初始化
  ├── 设备选择（CUDA / CPU）
  ├── 构建带自环的归一化稀疏邻接矩阵（用于散射）
  └── 实例化 Online / Target 模型 + Adam 优化器

  └── 每个 Epoch
        ├── train_online()
        │     ├── Online 前向：得到 h, h_pred, h_target
        │     ├── [每 gb_rebuild_every epoch] 构球 + 扩散 + 回写 → 新 h_pred

        │     ├── 球级 InfoNCE 损失（可选）

        │     └── 反向传播 + EMA 更新 target_encoder
        └── train_target()
              ├── Target 前向
              └── 防坍缩损失反向传播

评估
  └── 线性探测（LogisticRegression + GridSearchCV，3 次随机划分取均值）

写 CSV（results/<DATASET>_summary.csv）
```

#### 粒球触发逻辑

```python
if args.use_gb and (epoch % args.gb_rebuild_every == 0):

    # 并记录粒球统计到 granular_count/<dataset>.txt
```

## 4. 训练流程全链路

```
图数据
  │  x [N, F], edge_index [2, E], y [N]
  ▼
归一化邻接矩阵 slsp_adj（带自环，对称归一化）
  │
  ▼
┌─────────────────── Online 网络 ───────────────────────────┐
│                                                            │
│  GCN Encoder → h_1 [N, d]                                 │
│  散射（K 跳邻接传播）→ h_2 [N, d]                          │
│  h = h_1 + h_2   （全局感知嵌入）                          │
│                                                            │
│  ┌── 粒球模块（每 gb_rebuild_every epoch 执行）────┐       │
│  │  build_granules → GB_node_list, GB_center_list │       │
│  │  _build_ball_graph → W~ [B, B]                 │       │
│  │  _diffuse_on_ball_graph → HK [B, d]            │       │
│  │  回写 → z_new = α·h + (1-α)·HK[ball]  [N, d]  │       │
│  └──────────────────────────────────────────────  ┘       │
│                                                            │
│  predictor(z_new) → h_pred [N, d]                         │
│                                                            │
│  损失合成：                                                │
│    L = L_node（BYOL）                                      │
│      + λ1 · L_ball_scatter（RSM + 均匀）                   │
│      + λ2 · L_ball_infonce（匈牙利对齐）                   │
└────────────────────────────────────────────────────────────┘
  │
  ▼  EMA 更新
┌─────────────────── Target 网络 ────────────────────────────┐
│  GCN Encoder → h_target [N, d]                             │
│  防坍缩损失：L_target = -mean[(h - mean(h))²]              │
└────────────────────────────────────────────────────────────┘
  │
  ▼（训练完成）
线性探测评估
  └── 使用 h_1 + h_2（online embed）作为节点表示
      LogisticRegression + GridSearchCV（C ∈ 2^{-10..10}）
      3 次随机 20%/80% 划分取均值
```

## 5. 损失函数体系

### 5.1 节点级 BYOL 损失 `L_node`

```
z1 = normalize(h_pred)
z2 = normalize(h_target)  [stop gradient]
L_node = -mean(z1 · z2)   （余弦负相似度）
```

### 5.2 球级散射损失 `L_ball_scatter`

组合两项：

**散射项**（鼓励球间角度大于阈值）：
```
sep = mean[(1 - cos_sim(i,j)) × brake(i,j)]
brake = clamp(angle(i,j) / angle_thresh, max=1.0)
```

**均匀分布正则**（防止球心聚堆）：
```
uni = mean[exp(-2 × dist²(i,j) / tau_u)]
L_ball_scatter = sep + uni
```

### 5.3 球级 InfoNCE 损失 `L_ball_infonce`

```
正样本对：匈牙利匹配（Online 视图球 ↔ Target 视图球，最大化 Jaccard）
负样本：所有 Target 球

L_infonce = -mean[log(exp(pos/T) / Σexp(all/T))]
```

### 5.4 目标网络防坍缩损失 `L_target`

```
z = normalize(h_target)
L_target = -mean[Σ(z - mean(z))²]    （鼓励嵌入有方差）
```

## 7. 实验管理工具

### 7.1 参数扫描 `sweep.py` / `sweepX.py`

两阶段扫参框架（Stage-A 粗筛 + Stage-B 精训）：

| 阶段 | epochs | trials | 目的 |
|------|--------|--------|------|
| A（粗筛）| 150 | 1 | 快速定位优势参数组合 |
| B（精训）| 700 | 5 | 精确评估最终性能 |

**`sweep.py`**：简版，只搜索 `(quity, sim, alpha)` 三维空间  
**`sweepX.py`**：扩展版，全参数空间搜索（含扩散参数 + 球级损失参数），支持完整 12 元组候选

两者均支持：
- **断点续跑**：检查 CSV 是否已有该组合记录
- **多进程并行**：`ProcessPoolExecutor`，默认 2 workers
- **数据集别名**：CS/Co.CS/CoCS 等多种命名兼容

### 7.2 结果分析 `analyze_results.py`

```
输入：results/*_summary.csv
处理：
  ├── 过滤 use_gb=1 的行
  ├── 按超参组合分桶聚合（均值/标准差）
  └── 按均值降序排名，输出 Top-K

输出：
  ├── analysis/<DS>_gb_summary.csv  （每个数据集的组合排名）
  └── analysis/overall_topk.csv     （跨数据集汇总）
```
