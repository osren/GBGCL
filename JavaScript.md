##### 冒泡阶段

###### 是什么

####### 事件冒泡是指事件从最内层的目标元素开始，逐级向外传播，直到最外层的祖先元素（通常是 document 或 window）。在这个过程中，每个经过的元素都会触发其注册的同名事件处理程序

###### 作用

####### 事件委托：通过在父元素上监听事件，可以处理多个子元素的事件，从而减少事件监听器的数量，提高性能

####### 灵活性：可以在不同层级的元素上处理同一事件，提供更多的灵活性

####### 默认行为：浏览器的一些默认行为（如点击链接跳转）也是基于事件冒泡机制的

###### 这样一个结构我点击这个button，这个点击事件会从这个button进行冒泡，依次向父级元素冒泡，也就是说，如果不阻止所有的父级元素都会收到这个点击事件。

#### 阻止冒泡

##### 事件流动到这里执行之后就会停止冒泡，不再向上传递

#### 阻止默认行为

##### 在JavaScript中，可以通过调用事件对象的 preventDefault 方法来阻止默认行为。preventDefault 方法告诉浏览器不要执行与事件关联的默认动作

### 原型

#### 原型对象

##### 原型对象就是prototype，所有的构造函数和类都会有一个原型对象prototype用来存储一类实例应该共享的行为

##### 这里Person是一个构造函数，他可以理解成一个类，他有公共的属性name，以及挂在在prototype上的公共方法greet

##### 内存存储

###### Person 构造函数：是一个函数，位于内存中的一块区域。

###### Person.prototype：是一个对象，位于内存中的另一块区域。它保存着 greet 方法，所有通过 Person 创建的实例都会共享这个方法

###### new Person() 创建的实例：当你用 new 关键字创建一个新实例时（例如 const person1 = new Person("Alice");），该实例会有一个 __proto__ 属性，指向 Person.prototype。同时，实例会有自己的属性（如 name），这些属性是私有的，不会共享。

#### 对象原型

##### __proto__

###### 每一个对象都会有一个__proto__属性（包括prototype也会有这个属性），属性指向构造函数的 prototype 对象

###### 当你在实例上访问一个方法时，JavaScript 会首先检查该实例对象是否有该方法。如果没有，它会沿着原型链查找，直到找到或者到达原型链的末尾（null）

#### 原型链

##### 

##### 例子

###### 

#######  dog.__proto__ 指向 Dog.prototype。

####### dog.__proto__.__proto__ 指向 Animal.prototype

####### dog.__proto__.__proto__.__proto__ 指向 Object.prototype，这是所有对象的终点

####### 原型链查找过程

######## 首先，JavaScript 会检查 dog 对象本身是否有 speak 方法。因为 dog 实例本身没有定义 speak 方法，所以它会继续查找。

######## 然后，JavaScript 会查找 dog.__proto__（即 Dog.prototype）是否有 speak 方法。由于 Dog.prototype 没有 speak 方法，JavaScript 继续沿原型链向上查找

######## 接下来，JavaScript 会查找 Dog.prototype.__proto__（即 Animal.prototype）是否有 speak 方法。由于 Animal.prototype 上定义了 speak 方法，JavaScript 就找到并调用了该方法。

######## 最终，dog.speak() 输出 Buddy makes a sound.。

### 浅拷贝和深拷贝

#### 数据存储

##### 在js里面数据分为简单数据类型和复杂数据类型

###### 简单数据类型包括：number（数字）、string（字符串）、boolean（布尔值）、undefined、null、symbol。

####### 这些基本类型的数据是直接存储在栈内存中的，栈内存的管理非常高效。栈是由操作系统管理的一块内存区域，存储的是直接的值。每个基本数据类型的值会在栈中有一个固定的空间

####### 基本类型的变量在栈中存储的是值本身，不是地址

###### 复杂数据类型包括：object（对象）、array（数组）、function（函数）等。

####### 这些数据类型存储在堆内存中。堆内存不像栈那样按顺序分配和回收，它的内存分配更加灵活、复杂。堆内存用于存储动态分配的对象，而这些对象的引用会存储在栈中

####### 在堆内存中存储的是对象的实际数据，而栈中的变量存储的是指向该数据的引用（即内存地址）

####### 如果多个变量引用同一个对象，它们指向的是同一个堆中的内存区域

###### 

#### 

##### person是一个对象这个对象存储在栈上，占据了一块内存。person对象包含的这些数据(name，age，address)是存储在堆上的。

### 防抖和节流

#### 防抖

##### 核心思想

###### 防抖的核心思想是：事件触发后，设置一个定时器，只有在该事件触发结束后（即一定时间内没有再次触发该事件），定时器才会执行对应的操作。如果事件再次触发，则重新设置定时器

###### 

#######  let timer;: 用于保存定时器的 ID。每当事件触发时，我们都会创建一个新的定时器，并清除之前的定时器。

####### return function(...args): 这个函数是实际的事件处理函数，它会在事件触发时执行。...args 是 ES6 的扩展运算符，它表示收集所有传递给回调函数的参数。

####### clearTimeout(timer): 每次事件触发时，我们会先清除之前设置的定时器。这样，timer 保持为最新的定时器 ID。

####### timer = setTimeout(() => { fn.apply(this, args); }, delay): 在每次触发事件时，都会重新设置一个定时器，定时器会在 delay 毫秒后执行 fn 函数。

#### 节流 

##### 核心思想 

###### 节流是指在一定时间内，强制限制某个函数的执行次数。无论事件被触发多少次，只有在指定的时间间隔内才会执行一次回调函数。常见的应用场景有滚动监听、页面元素的拖动、点击按钮等。

####### url	用于 URL 输入，提供格式校验功能（如必须以 http:// 或 https:// 开头）。

####### search	用于搜索框输入，样式可能与普通文本框稍有不同。

###### 数值输入

####### number	用于数字输入，可以设置 min、max 和 step 属性。

####### range	用于滑块输入，用户通过拖动滑块选择一个数值范围。

###### 日期和时间输入

####### date	用于日期选择（仅支持日期，如 YYYY-MM-DD）。

####### time	用于时间选择（仅支持时间，如 HH:MM）。

####### datetime-local	用于日期和时间选择（不包括时区，如 YYYY-MM-DDTHH:MM）

####### month	用于月份选择（仅支持年月，如 YYYY-MM）。

####### week	用于选择某年的某一周（如 2023-W01 表示 2023 年第 1 周）。

######  选择和文件上传

####### file	用于文件上传，允许用户选择文件（可以设置 multiple 属性允许多选）。

####### checkbox	用于多选框，用户可以选择一个或多个选项。

####### radio	用于单选按钮，用户只能从一组选项中选择一个。

####### color	用于颜色选择，提供颜色选择器界面。

###### 按钮和提交

####### button	用于普通按钮，没有默认行为，通常与 JavaScript 配合使用。

####### submit	用于表单提交按钮，点击后提交表单。

####### reset	用于表单重置按钮，点击后重置表单字段为初始值。

####### image	使用图片作为提交按钮，功能与 submit 类似。

##### 只需要了解有这些输入的input类型，但是具体的细节可以在用到的时候学习
