## 一、粒球分裂阈值设计

### 1.1 分裂条件代码

**代码位置**：`granular.py:169`

```python
if quality_f > (qa + qb) / 2.5:
    # 停止分裂，保留原粒球
    split_GB_list.append(list(graph.nodes()))
else:
    # 继续分裂成两个子粒球
    split_bfs(sub_a, ...)
    split_bfs(sub_b, ...)
```

### 1.2 数学公式

$$\text{quality}_{parent} > \frac{\text{quality}_a + \text{quality}_b}{2.5}$$

### 1.3 设计原理

| 方面 | 说明 |
|------|------|
| **策略类型** | 贪心策略：只有分裂后质量不会显著下降时才分裂 |
| **阈值 2.5** | 父球质量需超过子球平均质量的 **2.5 倍** 才停止分裂 |
| **等价含义** | 分裂后质量下降超过 **60%** 才停止分裂（质量保留率 > 40%） |

### 1.4 阈值选择依据

- **阈值太大**（如 3.0）→ 分裂困难，粒球少且大，粒度过粗
- **阈值太小**（如 1.5）→ 过度分裂，粒球多且小，粒度过细
- **当前值 2.5** → 平衡粒度和质量的经验值

### 1.5 其他分裂保护条件

```python
# 1. 最小节点数保护
if node_num <= 3:
    return  # 太小的子图不再分裂

# 2. 最小粒球大小保护
if len(clusters[0]) < 2 or len(clusters[1]) < 2:
    return  # 分裂后粒球太小则不分裂

# 3. 连通性保护
if not nx.is_connected(sub_a) or not nx.is_connected(sub_b):
    return  # 分裂后不连通则不分裂
```

## 二、减少远距球与噪声传播的机制

### 机制一：KNN 稀疏化（球图构建）

**代码位置**：`gb_utils.py:136-141`

```python
# KNN 稀疏化（按中心相似度）
if knn > 0 and B > knn:
    _, topk_idx = torch.topk(sim_center, k=min(knn + 1, B), dim=1)
    mask_knn = torch.zeros_like(sim_center, dtype=torch.bool)
    for i in range(B):
        mask_knn[i, topk_idx[i]] = True
    sim_center = torch.where(mask_knn, sim_center, torch.zeros_like(sim_center))
```

**作用**：每个球只与最相似的 K 个球建立连接，**阻断远距球之间的信息传播**。

**参数**：
- `knn = 5, 10, 20`（在 sweepX.py 中搜索）
- 越小越稀疏，越能阻断远距传播

---

### 机制二：拓扑 + 语义融合（w_mode）

**代码位置**：`gb_utils.py:144-149`

```python
# 权重融合模式
if w_mode == "topo":
    W = topo_w              # 仅拓扑边
elif w_mode == "center":
    W = torch.relu(sim_center)  # 仅语义相似度
else:
    W = torch.relu(sim_center) + topo_w  # topo+center（默认）
```

| 模式 | 作用 | 特点 |
|------|------|------|
| `topo` | 只通过跨球拓扑边传播 | 更局部，信息传播受图结构约束 |
| `center` | 只通过语义相似度传播 | 语义相关但可能跨越远距 |
| `topo+center` | 两者融合 | **默认**，融合拓扑和语义，用 relu 去掉负相似度减少噪声 |

**关键设计**：`relu(sim_center)` 将负相似度置零，避免负相关传播带来的噪声。

---

### 机制三：β 控制扩散强度

**代码位置**：`gb_utils.py:168-169`

```python
# K 步扩散：H^{t+1} = (1 - β)H^t + β D^{-1}W H^t
for _ in range(K):
    H = (1 - beta) * H + beta * (P @ H)
```

**扩散公式**：

$$H^{(t+1)} = (1 - \beta) H^{(t)} + \beta \cdot D^{-1} W H^{(t)}$$

| 参数 | 作用 |
|------|------|
| β 越小（如 0.1） | 保留更多原始球心信息，扩散更保守，抗噪声能力强 |
| β 越大（如 0.3） | 扩散更激进，信息混合更充分，但可能引入噪声 |

**典型值**：β ∈ [0.1, 0.3]，默认 0.2

---

### 机制四：α 控制回写比例

**代码位置**：`gb_utils.py:199-204`

```python
# 回写：残差式融合
z_new = alpha_write * node_embed + (1 - alpha_write) * HK[b]
```

**回写公式**：

$$z_{new} = \alpha \cdot z_{orig} + (1 - \alpha) \cdot H_{K}$$

| 参数 | 作用 |
|------|------|
| α 越大（如 0.7） | 更多保留原始节点信息，减少噪声干扰 |
| α 越小（如 0.3） | 更多使用扩散后的粒球信息 |

**典型值**：α ∈ [0.3, 0.7]，实验发现 α=0.3 普遍最优

## 三、噪声传播抑制总结

| 机制 | 代码位置 | 作用 |
|------|---------|------|
| **KNN 稀疏化** | `gb_utils.py:136` | 只保留 K 个最近邻，阻断远距传播 |
| **w_mode 融合** | `gb_utils.py:144` | topo+center 融合，用 relu 去掉负相似度 |
| **β 扩散系数** | `gb_utils.py:168` | 控制扩散激进程度，越小越保守 |
| **α 回写比例** | `gb_utils.py:199` | 控制原始信息 vs 粒球信息的混合比例 |
| **最小粒球保护** | `granular.py:129` | 节点数 ≤3 不再分裂，避免过小粒球 |
| **连通性保护** | `granular.py:161` | 分裂后不连通则不分裂，保证粒球有效 |

## 五、设计思路总结

1. **分裂阈值 2.5**：通过经验值平衡粒球数量和语义一致性，避免过度分裂
2. **KNN 稀疏化**：从图层面阻断远距信息传播，只保留局部邻居
3. **w_mode 融合**：结合拓扑和语义，relu 激活函数去除负相关噪声
4. **β 参数**：控制扩散步幅，较小值更保守，较大值更激进
5. **α 参数**：控制最终融合比例，允许保留原始节点特征作为"锚点"

这套机制共同作用，使得粒球级别的扩散比传统节点级别扩散更具抗噪声能力。
